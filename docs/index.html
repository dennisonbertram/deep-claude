A different mind. The same harness.

deep-claude points Claude Code at DeepSeek's V4 models and redirects its state into a private home — so you get the full agentic harness on a different model, and your real Anthropic login is never touched.

  1. 1.0The idea.

    Claude Code is one of the best agentic coding harnesses there is — but the planning loop, tool use, MCP servers, sub-agents, permissions, and slash commands are all harness, not model. DeepSeek's V4 models speak the Anthropic API, so you can keep every bit of that machinery and just swap the brain.

    That means real workflows, not just chat: multi-step edits, test loops, MCP tools, and sub-agents all drive the same agent loop you already know — now thinking with DeepSeek V4. v4-pro is sharp on reasoning and long-context refactors; flash is fast and cheap for triage and bulk passes.

  2. 2.0The isolation.

    The friction was never the API — it's that a second model would collide with your normal Anthropic-authed Claude Code: shared session history, shared MCP config, shared ~/.claude.json. deep-claude sidesteps all of it. On each launch it sets four environment variables and redirects $HOME plus the XDG config / cache / data / state dirs into a private folder beside the repo:

    1. points ANTHROPIC_BASE_URL at api.deepseek.com/anthropic,
    2. passes your DEEPSEEK_API_KEY through as the auth token,
    3. selects the model (deepseek-v4-pro by default), and
    4. redirects $HOME into .deep-claude-home/, so all of Claude Code's state lives there instead of in your real ~/.claude/.

    Your normal setup is safe.

    The wrappers only flip environment variables per launch. Your Anthropic credentials are never read, copied, or modified — your everyday Claude Code stays exactly as it was.

  3. 3.0The setup.

    Three lines to install. install.sh symlinks deep-claude and deep-cco into ~/.local/bin — make sure that's on your PATH.

    sh 
    git clone https://github.com/dennisonbertram/deep-claude
cd deep-claude
./install.sh   # symlinks deep-claude + deep-cco into ~/.local/bin

    Then give it a DeepSeek API key. On macOS the Keychain is the clean path — the key never lives in .env, shell history, or process listings. (A shell export or a .env file work too.)

    sh 
    security add-generic-password -s deep-claude -a deepseek -U -w
# paste the key, then return — click "Always Allow" the first time

  4. 4.0The run.

    From here on, type deep-claude wherever you typed claude. Everything except --model passes straight through — flags, prompts, --resume, all of it. The harness is identical; only the model behind it has changed.

    console 
    $ deep-claude
…interactive session on deepseek-v4-pro (default)…

$ deep-claude --model flash -p "triage these failing tests"
…the agent loop, on the faster deepseek-v4-flash…

    Two model aliases, plus the full names when you want them:

    pro— the default
    deepseek-v4-pro — strongest reasoning, refactors, long context.
    flash
    deepseek-v4-flash — fast and cheap for triage, scripting, bulk passes.

    The wrapper eats --model X itself; use -- to be explicit when claude's own flags overlap — deep-claude --model flash -- -p "hello" is unambiguous.

  5. 5.0The sandbox.

    Letting an agent run with --dangerously-skip-permissions is wonderful right up until a prompt injection points it at your $HOME. deep-cco is the same wrapper, but everything runs inside cco's OS sandbox — Seatbelt on macOS, bubblewrap on Linux, Docker as a fallback. Same DeepSeek session history, walled off from the rest of your machine.

    console 
    $ deep-cco --model flash -p "refactor this module and run the tests"
◆ sandboxed  deepseek-v4-flash · cco
…full autonomy inside the sandbox, your $HOME untouched…

$ deep-cco --safe -p "hi"   # cco's tighter sandbox — hides $HOME

    --model is consumed locally; everything else flows to cco and then to claude. Both wrappers share the same .deep-claude-home/, so your DeepSeek history is consistent whether you run sandboxed or not.

Why run it.

Five reasons to keep the harness and swap the model.

Real workflows, not just chat.multi-step · MCP · sub-agents

Planning, editing, running tests, calling MCP tools, spawning sub-agents — the entire Claude Code agent loop works, now driven by DeepSeek V4. You're not dropping to a chat box; you're running the same autonomous coding sessions with a different model at the wheel.

Genuinely strong at code.v4-pro · v4-flash

V4-pro holds its own on reasoning, refactors, and long-context work; flash is quick and inexpensive for triage, scripting, and bulk passes. Pick the model to fit the job without leaving the tool.

Cost-effective by the token.DeepSeek API rates

Pay DeepSeek's API rates for heavy autonomous runs and long agent loops, while keeping your Anthropic subscription pristine for everything else. Route the expensive, exploratory work where it's cheapest.

Zero contamination..deep-claude-home/

Session history, projects, MCP config, and ~/.claude.json all live in a private home beside the repo. Your normal Claude Code setup is untouched — switch models without switching context, and without cross-talk.

Sandbox on demand.deep-cco

One command runs the whole thing inside a real OS sandbox via cco, so you can hand an agent --dangerously-skip-permissions without handing it your $HOME. Namespace isolation survives the sandboxing layer.

The reference.

The whole surface. Anything deep-claude doesn't recognize belongs to claude.

Run

deep-cco --safecco's tighter sandbox — hides $HOME.
